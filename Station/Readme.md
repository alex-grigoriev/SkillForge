# 🚉 Симуляция работы железнодорожного вокзала (C++)

## 📌 Описание проекта

Данный проект реализует **симуляцию работы железнодорожного вокзала** с использованием многопоточности в C++.

Несколько поездов одновременно движутся к одному вокзалу, однако:

> 🚧 В один момент времени на вокзале может находиться только **один поезд**.

Остальные поезда должны ожидать освобождения станции.

---

## 🎯 Цель задания

* Освоить **многопоточность (`std::thread`)**
* Научиться работать с **мьютексами (`std::mutex`)**
* Избежать **гонок данных (race conditions)**
* Смоделировать синхронизацию потоков

---

## 🚆 Участники симуляции

В программе участвуют **3 поезда**:

* 🚆 Поезд **A**
* 🚆 Поезд **B**
* 🚆 Поезд **C**

Каждый поезд:

* стартует одновременно (в отдельном потоке)
* имеет своё время в пути
* прибывает на вокзал независимо от других

---

## ⚙️ Логика работы

1. Пользователь вводит время в пути для каждого поезда (в секундах)
2. Все поезда начинают движение
3. По прибытии:

   * поезд пытается занять вокзал
   * если вокзал занят → ожидает
4. Когда поезд занимает вокзал:

   * программа ждёт команду `depart`
5. После команды:

   * поезд покидает вокзал
   * следующий поезд может зайти

---

## 🔒 Синхронизация

Для контроля доступа используется:

```cpp id="mxg7p9"
std::mutex station_mutex;
```

### Поведение:

* `lock()` — поезд занимает вокзал
* `unlock()` — освобождает вокзал
* остальные потоки блокируются до освобождения
