# 🌳 Village Tree (C++)

Консольное приложение на **C++**, моделирующее деревья с ветками и "домиками" для эльфов.

Пользователь заполняет структуру деревьев, после чего может выполнить поиск по имени и получить информацию о расположении.

---

## 🚀 Возможности

- 🌳 Генерация нескольких деревьев
- 🌿 Случайное количество веток
- 🧝 Назначение имён "эльфов" для каждой ветки
- 🔍 Поиск по имени
- 📊 Вывод информации о расположении (дерево, ветки)

---

## 🏗️ Архитектура

Проект состоит из трёх частей:

```
main.cpp → точка входа, поиск
village.h → объявления классов (Tree, Branch)
village.cpp → реализация логики
```

---

## 📦 Структура данных

### 🌿 Branch

Содержит:
- имя (`name`)
- номер дерева (`tree`)
- верхнюю ветку (`branch1`)
- среднюю ветку (`branch2`)

---

### 🌳 Tree

Содержит массивы указателей:
- `nameAt[]`
- `treeAt[]`
- `branch1[]`
- `branch2[]`

Каждый индекс — отдельная "ячейка" (позиция эльфа в дереве)

---

### Сборка

```
./village
```

## 💻 Использование

**Программа создаёт деревья**

Для каждой ветки вводится имя эльфа:

```
_____1 tree
Top Branch count: 3
1 top branch: 2 middle branch count
Input 1 top branch house elf name: Legolas
Input 1 middle branch house elf name: Elrond
```

После заполнения:

```
Input name for search:
```
Ввод имени:

```
Legolas
```

Вывод:

```
Tree: 1 Top branch: 1 Middle branch: 0 Elf: Legolas
```

## 🧠 Особенности реализации
- Используется динамическая память (new)
- Хранение через массивы указателей (Branch**)
- Поиск выполняется линейно (O(n))
- Генерация веток через rand()
- Связь данных реализована через индексы

💡 Возможные улучшения
- 🔧 Заменить new[] на std::vector
- 🧹 Реализовать деструктор (освобождение памяти)
- 🎯 Использовать структуру вместо 4 массивов
- 🔍 Оптимизировать поиск (например, через map)
- 🎲 Вынести srand() вне цикла
- 🧱 Сделать нормальную иерархию: Tree → Branch → SubBranch
