# 🏠 Умный дом (Симуляция на C++)

## 📌 Описание проекта

Данный проект представляет собой упрощённую модель системы **умного дома для дачи**, реализованную на языке C++.
Программа симулирует работу дома в течение **двух суток (48 часов)**, принимая входные данные от пользователя каждый час и автоматически управляя устройствами.

---

## ⚙️ Функциональность

В системе используются следующие датчики:

* 🌡 Температура внутри помещения
* 🌡 Температура снаружи
* 🚶 Наличие движения снаружи
* 💡 Состояние света внутри дома (ручное управление)

---

## 🔌 Управляемые системы

Умный щиток управляет следующими устройствами:

* 💡 Освещение внутри дома
* 🌳 Освещение в саду
* 🔥 Отопление помещения
* 🚿 Обогрев водопровода
* ❄️ Кондиционер

---

## 🧠 Логика работы

### 🚿 Обогрев водопровода

* Включается: если температура < 0°C
* Выключается: если температура > 5°C

### 🌳 Уличное освещение

* Включается: если

  * есть движение
  * и время **с 16:00 до 05:00**
* Выключается: если нет движения или не вечер

### 🔥 Отопление

* Включается: если температура внутри < 22°C
* Выключается: если температура ≥ 25°C

### ❄️ Кондиционер

* Включается: если температура ≥ 30°C
* Выключается: если температура ≤ 25°C

---

## 💡 Освещение внутри (цветовая температура)

Если свет включён:

* В 00:00 → устанавливается **5000K**
* С 16:00 до 20:00 → плавное изменение:

  * от **5000K → 2700K**
* В остальное время температура не меняется

---

## ⏱ Формат работы программы

Каждый час пользователь вводит данные в формате:

```
<temp_inside> <temp_outside> <movement> <lights>
```

### Пример:

```
23 -5 yes on
```

### Где:

* `movement`: `yes` / `no`
* `lights`: `on` / `off`

---

## 📤 Вывод программы

Программа выводит:

* Только **изменения состояния устройств**
* Текущую цветовую температуру, если свет включён

### Пример вывода:

```
Heaters ON!
Garden lights ON!
Color temperature: 5000K
```

---

## 🧩 Технические особенности

* Используются **битовые флаги (enum)** для управления состояниями
* Ввод обрабатывается через `stringstream`
* Время моделируется циклом (48 итераций)
* Состояния устройств не дублируются (нет лишних сообщений)
